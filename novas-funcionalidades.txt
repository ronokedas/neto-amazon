# CONTEXTO GERAL

Você é um assistente especializado no desenvolvimento do sistema ERP da empresa
Amazon Naval Ltda, uma empresa de certificação naval localizada em Belém/PA.

O sistema é uma aplicação web desenvolvida em:
  - Backend:    PHP 8.2+ com PDO
  - Frontend:   HTML5, CSS3, JavaScript vanilla
  - Banco:      MySQL/MariaDB (chaves primárias UUID)
  - PDF:        Biblioteca FPDF
  - Segurança:  password_hash/verify, CSRF tokens, sessão 30min, Prepared Statements
  - Infra:      Docker (docker-compose)
  - Design:     Tema escuro, verde (#2ECC71), sidebar retrátil, Font Awesome 6, Inter

O sistema roda localmente em: http://localhost:8080/
Caminho físico no Windows: C:\sistema\

# LIMITAÇÕES ATUAIS (O QUE FALTA OU ESTÁ INCOMPLETO)
================================================================================

PROBLEMA 1 — Módulo de Pessoas não serve como "Cadastro de Clientes":
  O módulo atual armazena apenas dados básicos (nome, CPF, telefone, email).
  Falta: distinção de perfil (Armador / Proprietário / Despachante), CNPJ,
  vínculo com embarcações e dados comerciais.

PROBLEMA 2 — Módulo de Embarcações não está vinculado a clientes:
  Não existe relação entre embarcação e cliente/armador no banco de dados.
  Os campos atuais são: nome, registro, tipo, ano, proprietário (texto livre),
  observações. Faltam dados técnicos completos da embarcação.

PROBLEMA 3 — Módulo Comercial inexistente:
  Não existe geração de proposta comercial no sistema. O orçamento hoje é
  gerado manualmente por planilha Excel e convertido em PDF fora do sistema.

PROBLEMA 4 — Módulo de Agendamento / Ordem de Serviço inexistente:
  Não existe agendamento formal de vistorias com OS vinculada.
  O módulo de vistorias atual é apenas um registro simples.

PROBLEMA 5 — Módulo de Vistoria atual é insuficiente:
  O vistoriador não recebe agendamentos automaticamente.
  Não existe geração de relatório de vistoria dentro do sistema.
  Não existem campos para exigências e observações técnicas.

PROBLEMA 6 — Certificados limitados ao CSN:
  O sistema gera apenas o Certificado de Segurança da Navegação (CSN).
  Faltam: CNBL (Borda Livre), CNARQ (Arqueação), e a Proposta de Serviço/
  Orçamento em PDF.

PROBLEMA 7 — Numeração sequencial manual:
  Os números de certificados, relatórios e orçamentos são inseridos à mão.
  O sistema deve gerar sequenciais automáticos por tipo de documento.

--- C. MÓDULO COMERCIAL — PROPOSTA / ORÇAMENTO ---

  Subseção: Proposta de Serviço

  Fluxo ao criar nova proposta:
    1. Usuário clica em "Nova Proposta"
    2. Sistema exibe lista de clientes cadastrados para seleção
    3. Após selecionar o cliente, sistema carrega automaticamente as
       embarcações vinculadas a ele
    4. Usuário seleciona a embarcação desejada
    5. Sistema carrega os dados técnicos da embarcação automaticamente
    6. Usuário seleciona os serviços a realizar (checkboxes com preços)
    7. O sistema calcula total, desconto e total geral automaticamente
    8. Geração do PDF no modelo Amazon Naval (igual ao exemplo-pdf-ORÇAMENTO)

  Serviços disponíveis (com preço cadastrado no sistema):
    - Análise de Planos (Ec1)
    - Análise de Planos (Ec2)
    - Vistoria Inicial Seco
    - Vistoria Inicial Flutuando
    - Vistoria Inicial de Borda Livre
    - Vistoria Inicial de Arqueação
    - Acompanhamento de Ultrassom
    - Vistoria Anual
    - Vistoria Anual Periódica
    - Vistoria Intermediária
    - Licença Provisória

  Numeração automática:
    - Formato: AM-ORC:{sequencial}/{ano} (ex: AM-ORC:11/26)
    - Sequencial incremental por ano, reinicia a cada ano

  Forma de pagamento padrão:
    - 3 parcelas iguais (editável)
    - Dados bancários fixos da empresa (configurável em config.php)

  Por embarcação:
    - Cada embarcação deve ter seus serviços listados separadamente
    - Subtotal por embarcação + Total geral da proposta

  Indicadores comerciais (para o ADMIN):
    - Número de propostas geradas no mês
    - Valor total de propostas no mês
    - Percentual de meta atingida (meta configurável)

  ARQUIVOS A CRIAR:
    modules/comercial/index.php         ← listar propostas
    modules/comercial/nova.php          ← wizard de nova proposta

--- H. MÓDULO DE ENVIO DE EMAILS ---

  Por que é necessário:
    Hoje todo envio de proposta, certificado e link de assinatura é feito
    manualmente fora do sistema (WhatsApp, email avulso). Com o módulo de
    email integrado, o fluxo fecha 100% dentro do ERP: o usuário gera o
    documento e já envia ao cliente com um clique, sem sair do sistema.

  Tecnologia recomendada:
    - Biblioteca PHPMailer (via Composer: composer require phpmailer/phpmailer)
    - Protocolo SMTP com suporte a TLS/SSL
    - Configuração de credenciais SMTP em config.php (host, porta, usuário,
      senha, remetente padrão) — nunca hardcoded nos arquivos de módulo

  Configuração em config.php (adicionar):
    define('MAIL_HOST',       'smtp.gmail.com');   // ou outro SMTP
    define('MAIL_PORT',       587);
    define('MAIL_USERNAME',   'contato@amazonnatal.com.br');
    define('MAIL_PASSWORD',   'senha_app');         // senha de app Gmail
    define('MAIL_FROM_NAME',  'Amazon Naval Ltda');
    define('MAIL_ENCRYPTION', 'tls');

  Função central a criar em includes/functions.php:
    function enviarEmail(
        string $destinatario,
        string $nomeDestinatario,
        string $assunto,
        string $corpoHtml,
        array  $anexos = []   // [['path' => '/tmp/cert.pdf', 'nome' => 'CSN.pdf']]
    ): bool
      → retorna true/false e registra resultado em email_logs

  ONDE O EMAIL SERÁ USADO (pontos de disparo):

  H1. Proposta Comercial (módulo comercial)
      Gatilho: botão "Enviar Proposta" na tela de detalhes da proposta
      Destinatário: email do cliente cadastrado
      Anexo: PDF da proposta (AM-ORC)
      Assunto: "Proposta de Serviço Amazon Naval — {nome da embarcação}"
      Corpo: template HTML com dados da proposta e dados bancários para pagamento

  H2. Confirmação de Agendamento (módulo agendamentos)
      Gatilho: automático ao salvar um novo agendamento com status "Confirmado"
      Destinatário: email do cliente
      Sem anexo
      Assunto: "Vistoria Agendada — {embarcação} em {data}"
      Corpo: data, hora, local, tipo de vistoria, nome do vistoriador,
             contato para dúvidas

  H3. Envio de Certificados (módulo documentação)
      Gatilho: botão "Enviar Certificado" após emissão de CSN, CNBL ou CNARQ
      Destinatário: email do cliente
      Anexo: PDF do certificado correspondente
      Assunto: "Certificado {tipo} Emitido — {embarcação}"
      Corpo: dados básicos do certificado, validade, orientações sobre
             convalidações anuais

  H4. Link de Assinatura Digital (já existe no CSN, generalizar)
      Gatilho: botão "Enviar para Assinatura" em qualquer certificado
      Destinatário: email do responsável pela assinatura
      Sem anexo
      Assunto: "Assinar Documento — Amazon Naval Ltda"
      Corpo: link único de assinatura + instruções + prazo de validade do link

  H5. Alerta de Vencimento de Certificados (automático / cron)
      Gatilho: script de verificação diária (via cron job ou chamada agendada)
      Arquivo: scripts/alerta_vencimentos.php
      Lógica: busca certificados com validade nos próximos 30 e 7 dias
      Destinatário: email do cliente vinculado à embarcação
      Assunto: "⚠️ Certificado vencendo em {N} dias — {embarcação}"
      Corpo: nome do certificado, data de vencimento, instrução para contato

  HISTÓRICO DE EMAILS (rastreabilidade):
    Toda vez que um email for enviado, registrar na tabela email_logs para
